# **Restaurant Admin App - Comprehensive Overview & Rules**

## **🏪 Business Context**
You operate a restaurant with **4 Ghost Kitchen Brands**:
1. **Crispy Chicken Lab** - Chicken-focused menu
2. **Devils Smash Burger** - Burger-focused menu  
3. **The Bowl Spot** - Bowl/healthy food menu
4. **Tacotastic** - Mexican/taco menu

## **🔄 Core Business Flow & Data Management**

### **📋 ORDER MANAGEMENT SYSTEM**

#### **Two Ways Orders Enter the System:**

**1. Manual Order Creation** (`lib/widgets/cart_view_widget.dart`)
- Staff manually creates orders through cart interface
- Select brand, menu items, quantities, pricing
- Choose order type (delivery/pickup), payment method
- Enter customer details for delivery orders
- **Inventory Impact**: Automatically subtracts materials when order is confirmed

**2. Automated Receipt Scanning** (Fujitsu ix100 Scanner)
- **Hardware**: Fujitsu ix100 scanner for physical receipts
- **Local Server**: `lib/services/local_scan_server.dart` receives scanned images
- **AI Processing**: Supabase Edge Function (`supabase/functions/scan-and-create-order/index.ts`) uses Gemini AI
- **Data Extraction**: AI extracts order details, items, prices, delivery fees, customer info
- **Database Insertion**: Automatically creates order with `pending_confirmation` status
- **Auto-Confirm Dialog**: `lib/widgets/auto_confirm_order_dialog.dart` shows 7-second countdown
- **Inventory Impact**: Materials subtracted after confirmation

#### **Order Status Flow:**
```
Scanned Receipt → pending_confirmation → (Auto-confirm/Manual confirm) → confirmed → preparing → completed
Manual Order → confirmed → preparing → completed
```

#### **Real-Time Order Updates:**
- **GlobalOrderListener** (`lib/widgets/global_order_listener.dart`) - Monitors new orders globally
- **Orders Screen Subscription** - Real-time updates when orders change
- **Auto-popup for pending orders** - Shows confirmation dialog immediately

---

### **📦 INVENTORY MANAGEMENT SYSTEM**

#### **Material Items** (`lib/models/material_item.dart`)
- **Core Properties**: ID, name, current quantity, unit of measure, weighted average cost
- **Tracking**: Seller info, item numbers, categories, notification thresholds
- **Storage**: Local Hive database for offline capability

#### **Inventory Updates:**
**Automatic Subtraction (Orders):**
- When orders are confirmed, materials are automatically subtracted
- Based on menu item recipes (`lib/models/menu_item_material.dart`)
- Real-time quantity updates across all screens

**Manual Addition (Purchases) - PLANNED:**
- Scan wholesale purchase receipts
- AI extracts material items, quantities, costs
- Automatically adds to inventory
- Updates weighted average costs

#### **Menu Item Materials** (`lib/models/menu_item_material.dart`)
- Links menu items to required materials
- Defines quantities needed per menu item
- Enables automatic inventory deduction

### **🚚 DELIVERY MANAGEMENT SYSTEM**

## **🎯 Key Business Rules & Requirements**

### **Inventory Management Rules:**
1. **Orders SUBTRACT materials** - When confirmed, automatically reduce inventory
2. **Purchases ADD materials** - When scanned, automatically increase inventory  
3. **Real-time Updates** - Inventory must always reflect current state
4. **Weighted Average Costing** - Track material costs accurately
5. **Low Stock Alerts** - Notify when materials reach threshold levels

### **Order Processing Rules:**
1. **Dual Input Methods** - Manual cart + automated scanning
2. **Auto-Confirmation** - 7-second countdown for scanned orders
3. **Real-time Notifications** - Immediate popup for new orders
4. **Status Tracking** - Clear progression from pending to completed
5. **Delivery Integration** - Seamless handoff to delivery system

### **Scanner Integration Rules:**
1. **Two Scan Types** - Orders (subtract inventory) vs Purchases (add inventory)
2. **AI Processing** - Gemini extracts structured data from receipts
3. **Error Handling** - User can improve prompts when AI fails
4. **Immediate Processing** - Fast kitchen workflow, minimal delays

### **Multi-Platform Compatibility:**
1. **Mobile-First Design** - Optimized for phone usage
2. **Cross-Platform** - Flutter ensures iOS/Android compatibility
3. **Offline Capability** - Local Hive storage for inventory
4. **Real-time Sync** - Supabase handles live updates
